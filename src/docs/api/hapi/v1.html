hAPI

The scope of Hopster API is to:

  • Manage users and their carts, identity and attendant metadata

  • Generate a list of offers available to the current user

  • Inject a print experience into a remote page

  • Provide a mechanism to perform activities and unlock coupons

High level use

  • create a new Session, or new User (which will come with a new session)

  • get Offers

  • unlock Offers by completing activities

  • add offers to the Cart associated with the Session

  • print Offers

Session 

A session can be created for one of the following types of identities:

  • Anonymous users, type=anonymous - A user who is not authenticated and whose session is identified by an unique token (uuid)

  • Partner users, type=partner - Similar to anonymous users, but a partner/brand supplies the token

  • SMS users, type=sms - A user who is authenticated via SMS and identified by their phone number

  • Facebook users, type=facebook - A user who is authenticated by Facebook and identified by their Facebook user id

  • Hopster users, type=email - A user who is authenticated by password and identified by email

Current Session 

A session can have the following attributes:

  • uuid (string) - A uuid is always present.

  • type (string) - One of the allowable session types

  • email (string) - if type is email

  • phone (string) - if type is sms

  • facebook_id (number) - if type is Facebook
